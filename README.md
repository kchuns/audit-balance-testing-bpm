# Audit Balance Testing BPMN Project

## Overview
This project analyzes the process of audit balance testing performed by an audit intern during an audit engagement. It models how work flows between the audit intern, client, and senior auditor using BPMN.

## The Problem
Audit testing often involves delays due to incomplete client documentation, back-and-forth communication, and revisions during review. These inefficiencies can slow down the audit process.

## Process Modeled
The process begins when a senior auditor assigns a testing area. The intern requests supporting documentation from the client, reviews it, compares it to the general ledger, and documents findings. The senior auditor then reviews the work and either approves it or sends it back for revision.

## Key Findings
- Client delays create bottlenecks
- Incomplete documentation causes rework
- Senior review can slow down final completion

## Recommendations
- Use a standardized support checklist
- Improve document organization
- Automate follow-ups for missing support

## What I Learned
This project helped me understand how audit work can be viewed as a process with clear steps, decision points, and inefficiencies. BPMN made it easier to visualize delays and loops in the workflow.

## Who Would Care
This would be useful for audit firms, interns, and managers looking to improve workflow efficiency and reduce delays in audit testing.
